菜鸟裹裹使用体验

Posted on 2017-02-24 | | Visitors

最开始的菜鸟裹裹只是查询物件信息以及查看附近快递员信息，目前已增设了上门取件，自提点寄件，校园包裹侠等功能，同时在物件运输方面遵循菜鸟网络更快更近的原则。

菜鸟网络是指（个人理解）：通过各物流系统依据统一规则管理物件，做到快递单电子化的同时实现强大的物件调度管理，从而构成了一个诺大的物流网络。

上门取件 / 自提点寄件

功能背景 / 寄件场景

当前寄件的方式主要有：去快递站点或代理点寄件、联系熟悉的快递员上门取件、放到相关的自提点寄件。

一、去快递站点或代理点寄件

  • 优点
    • 去到填完邮单就可以离开（邮单可能是电子邮单，直接在手机上完成邮单信息填写并付费）
    • 避免物件不可寄被通知退回，可及时获取并解决
    • 对于邮单上材料填写有疑问的可及时询问工作人员并得到答复 比如物件类型要填写什么
  • 缺点
    • 快递站点或者代理点离得远，跑一趟花时间

二、联系熟悉的快递员上门取件

  • 优点
    • 不需要跑一趟直接有人上门
    • 交代熟悉的快递员寄件会比较放心和舒心
  • 缺点
    • 快递员到来的时间不可估算 有时候需要等待较长时间

三、放到相关的自提点寄件

  • 优点
    • 直接放到附近的自提点后便可离开
  • 缺点
    • 自提点当前覆盖的城市及区域有限，在短期内来说只是部分城市部分市民能够受益

功能看法

从上面的寄件方式及其优缺点可了解到，目前寄件存在的主要问题是：

  • 去快递点寄件个人时间内耗成本较高
  • 上门寄件可能联系不上快递员或快递员迟迟未到
  • 自提点寄件但是周围没有设置自提点

所以可否做这样的一个假设：当上门寄件的速度足够快以及设置的自提点足够多的情况下，去快递点寄件的寄件方式就会被淘汰，而快递点和寄件人建立的关系都是【线上关系】，来往会更加便利。

那么上门寄件的速度要如何提升以及自提点的设置应该如何规划呢？

目前菜鸟裹裹采用的是快速匹配周围快递员或者是最近的快递点的办法来解决上门寄件速度的问题，以快速匹配周围快递员为主。并且将寄件的人群主要锁定为二手货寄件、商品退货寄件及个人物件，个人物件的话其实是写字楼的量会相对大些。

建议 / 想法

所以以上问题就转换为如何能快速匹配上快递员并保证其在短时间内能够到达，其中包括匹配最近的快递点。其实这个问题就很像打车，我是随时随地都能快捷打到车而不是去到某个站点才能够拦截到车。所以快速匹配并在短时间能够完成的问题其实很像滴滴打车中怎么样能够给打车的人快速匹配到车并在短时间内成功搭乘。从这点来看，滴滴打车虽然与菜鸟裹裹的业务内容不同但是也是值得了解和分析的竞品之一。

校园包裹侠

功能背景

包裹侠是指那些接收【帮取包裹】请求并将包裹按时送达请求者手上的人，并会在这个过程中赚取一定费用。

至于在菜鸟裹裹中为什么是校园包裹侠不是职场包裹侠或者是包裹侠……我刚开始也不是很理解，就简单觉得可能菜鸟锁定了学生（大学生）这个群体吧。

快递的主要取件方式有：到快递点或者是代理点取件、快递员上门送件、自提点取件。而会发出想要帮取包裹请求的大体可以分成两种情况：需要去快递点或者代理点取但是不想去或者没时间去和快递员上门送件恰好不在。

针对后面一种情况：一般来说，收件人都掌握着物件的配送信息，能大概估算快递员上门，从而对自己的行程进行调整或安排，或者致电快递员提前沟通，这是比较理想的状态。当然，如果刚好接到电话自己又恰好不在，可以经协商后选择他人代签或者是由快递员在第二天进行配送。所以，在快递员上门配送存在的问题只是快递员送货上门但是恰好我不在，时间方面的冲撞问题，且也是有了较好的解决方式，需要的就是引导收件人合理选择收件时间（或者是寄件人合理选择配送时间），使得配送更高效。

而如果是前一种情况：需要去快递点或者代理点取件，导致“无法”出门去取，除了会有时间方面的冲突之外，还有就是懒得去取件，或者是行动不便等导致想要他人代劳。这个时候包裹侠就被号召而来了，而其实快递代取只是跑腿请求中的一个分支。那为何快递代取主要锁定了在了校园呢？

一般来说快递的派送都是会快递人员送货上门，但是对于校园以及超过配送范围地偏远地区就会需要收件人到某一快递点或者代理点进行取货，而这两大类人群中，校园的包裹数额大且校园里的收件人群性格特征明显，所以我想这也是菜鸟裹裹锁定校园包裹侠而不是职场包裹侠或单单只是包裹侠的原因，校园包裹侠让快递代取在菜鸟裹裹中的定位更加明显和清晰，且主要是锁定了大学校园。

校园里的收件人群性格特征：习惯使用新的方式让自己的生活更加便利，更愿意接受付费跑腿的人群性格。

功能看法

校园包裹侠锁定了大学校园的人群并抓住部分人群无法去快递点或代理点取快递的痛点，设置了【帮我取包裹】、【帮我跑跑腿】、【跑腿抢单】等功能，提供了快递代取和零碎时间赚取外快的平台。

那么在帮我跑腿和我帮你跑腿之间用户最关注的是什么呢？

无论是请求帮忙还是包裹侠都需要填写个人信息，而包裹侠需要填写的信息会更加详细，这里要考虑到：

  • 信息的真实性（信息验证）
  • 信息的全面性（出现包裹失踪能够找到该包裹侠，包裹迟迟未被送达能联系上包裹侠等）
  • 建立诚信评判标准（抢单的包裹侠能看到他的诚信情况，是否被投诉过等）这个暂未在菜鸟裹裹中发现

包裹侠可以通过抢单的方式进行抢截包裹，同时包裹的信息会注明是包裹的大小、预期送达的时间和地址以及付费的金额（当前全有用户填写）。

建议 / 想法

用户创建找人代取快递包裹的详细信息是根据自主编辑或是粘贴快递短信（有时需做删减）的方式进行补充的，但如果可以直接导入在菜鸟中的包裹信息，用户就省去了输入包裹详细信息这一步骤了，也不需要选择包裹的尺寸等信息（系统可以进行判断），只需要输入预期送达的时间和打算付费的金额即可。

当前创建代取的方式，感觉如同再填了一次快递单般不简单。

其它功能

寄件地址

选择寄件地址时会有一个地址簿，上面有淘宝的常用地址，能够满足自身对常用地址的需求，使得地址能够快捷输入。

而为什么采用定位锁定地址的同时，还要增加一个地址簿呢？个人感觉是菜鸟裹裹的另一用心之处，寄件不一定是寄送自己的物件，可能是代为寄送，所以就会要输入其它地址，而我们会去帮忙寄件的人绝大多数来说都是跟我们生活紧密的人，如家人、闺蜜等，我们可能会给她们购置过商品物件，故淘宝上备注的常用地址又能够满足和帮助到该需求。

导入包裹

能够登录其他常用的网购平台不需查询订单就能轻松了解包裹信息，从而也清楚哪些平台上购置的商品能够做到。

成为包裹侠

一、设置层级较深，难以发现
成为包裹侠（我发现的成为包裹侠的唯一入口）的位置放在了校园包裹侠-订单之后与【帮我取包裹】同级，但是点击进入订单是先看到【我发起的订单】和【帮我取包裹】，【成为包裹侠】需要点击【抢的任务】才能看见，所以实际【成为包裹侠】的层级比【帮我取包裹】的要低。

但是实际在进入校园包裹侠后就有【帮我取包裹】和【帮我跑跑腿】，而进入订单之后又有【帮我取包裹】但是【成为包裹侠】被搁置了，同时看到的是【抢的任务】的按钮，所以知道点击【抢的任务】会有【成为包裹侠】这个入口显然是比较困难的。

二、填写信息并进入下一步屡次失败

虽然填写包裹侠信息以及提示了建议在有WiFi的情况下进行填写，就告诉用户请保持网络通畅的情况下进行。我是使用WiFi的情况下进行填写的，但是每次到上传图片及上传学生证时就卡壳，点击上传之后并没有图片上传情况的信息反馈，直到最后上传完学生证后一段时间也未上传成功。

同时，在填写学号信息的时候点击输入框，弹出的键盘会遮挡输入框（使用讯飞输入法是如此，切换了系统自带输入法也是，手机型号：MX5）。

期望

随着时代的发展、人们消费主要模式的转换以及快递物流的发达，菜鸟裹裹是一个被赋予生命力的产品。校园包裹侠再一次带来了零碎空闲时间也能赚钱的渠道，同时快速匹配快递员和自提点寄件的方式能够让寄件人更加方便地寄送物件，在一定程度上解决了寄送痛点。虽然现在仍有一定的不足，如附件没有快递员或快递员不愿意接单、自提点的覆盖面积仍然较小等，但是恰恰也是被需要了，才会被提出并且希望得到改善！

期待菜鸟裹裹越来越好 ❤️

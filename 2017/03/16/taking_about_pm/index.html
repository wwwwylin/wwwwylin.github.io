人人都是产品经理吗？

Posted on 2017-03-16 | | Visitors

这个已经被说烂的话题我也试着说说…ヽ(*。>Д<)o゜one… two… three!!!

先前听刘飞的一个live关于产品经理求职的，里边他提到了一个现象就是，都说自己适合做产品经理

我觉得我很适合做产品经理啊，我看了很多关于产品的文章，觉得自己挺能够胜任的…

我对APP的功能有很多想法，我经常给应用商店提供建议…

我用过很多APP并且都能记得哪个类型的APP都有哪些，而且我把它们的主要功能都记录下来了…

但是你只要问他们比较深度的问题他就会支支吾吾说不出一个所以然。

也就是理论或是材料好像积攒了很多，但是当真要解决问题的时候却不知道如何下手。

你说你能胜任做产品经理，那你能举出一个例子证明吗，或者你觉得自己具备哪些特质所以觉得自己合适？

那你针对我们的“某一功能”有什么见解呢？如果让你负责一个关于【在高速路上】使用的导航产品，你会怎么设计？

你选一个你熟悉的类型并说出这个类型的产品的买点是什么，解决了什么痛点，它日后的发展会是怎样？

所以，怎样才能胜任产品经理的工作呢？

先来看一则问题：应届生直接做PM（产品经理）到底合不合适？

俞军老师在文中讲到：产品经理是要看天分的，有天分就合适，没天分就不合适。而有PM天分的人通常会有以下特征：

  • 逻辑严谨
  • 对细节敏感
  • 看问题视角全面
  • 总是能看到事物本质
  • 用户感：总是能准确察觉他人心理

在我经历了匣子开发之前，我觉得看到这些所谓的特征都觉得很虚，好像自己思维逻辑还蛮不错的，好像自己蛮注重细节，好像自己蛮能够观察别人的心理，但是这样的软实力跟产品的开发有半毛钱关系？软实力能够变成产品吗？

但是当开发了匣子之后，我觉得这样的特质是无形推动你产品的完成的，也许技术实现的不是你，视觉设计的不是你，但是协调并推进整一个产品的开发，参与整一个产品开发到上线到迭代的是你，而没有这些特质你很难推动产品开发。当然还有需求分析和设计等，也是需要这些特质。

然后发现曾以为的熟悉掌握原型软件很重要啊、数据分析能力很重要啊、时刻有着新的点子很重要啊等都其实是抓错了重点。同时，虽然在书本里面学习到了软件开发是要经过步骤：原型设计、视觉设计、产品开发、产品测试和上线，但是原型设计可能会在产品开发返工啊，原型设计之前最好是做信息架构搭建啊，信息架构搭建可以成为产品测试的依据啊，这个时候进行视觉评审的内容应该到什么程度，书本或是文章当中是不会明明白白写给你的，即便是有，不同的产品会有不同的开发模式及不同的面对人群等，一旦把一个方法写死了，这个方法就没有意义了，因为它失去了最基本的适用性。所以其实产品经理很重要的一个能力是，针对当前问题解决问题的能力，就是所具备的特质是通用的而不是死板的。

针对当前问题解决问题的能力就是要善于发现问题，抓住问题的核心并且能够在某一标准下采取方式解决问题。当然问题的发生是在产品经理的工作范围中的。

善于发现问题 = 勤于思考、对细节敏捷、思维谨慎…

抓住问题核心 = 总能够看到事情本质、看清局势集中力量干大事…

沟通能力 = 能否让不同角色的人都认识到问题（工作），并共同朝着同一方向奋斗，义无反顾…

同理心 = 能准确觉察他人心理的同时，能够感受到他的感受，明白他的痛点，跟他站在同一线上…

思维逻辑能力 = 思维严谨、思维分析能力强…

然后具备了这些能力（可能还有其他hhhh…我觉得主要就这几个吧）之后不管你原型是要用Axure画还是直接上纸画，你的思想已经在了，你的产品功能已经确定，信息架构已经搭建完善；不管公司具体的开发流程不同，但是你参与了一次完整的开发，已经具备会运用了通用的软实力；… …

所以也就有了俞老师概括的合适的产品经理需要具备的特质有哪一些。这些看似对开发没有实际作用的软实力确实作为产品经理不可缺少的应该具备的特质。

也许你会说，懂技术那不是更好吗，是啊懂技术当然好啊，它是加分项啊，你懂技术的话你跟开发小哥就能够更好地沟通，能更好更快地抓住他说的重点然后揪出问题核心呀。

也许你会说，能够熟练使用Axure不好吗，是啊能够熟练使用Axure是不错啊，但是如果你只是熟练地使用一个死的软件，而没有搭建好你原型的信息架构，后头软件使用时各种问题呢？所以，如果你在思维方面都已经很优秀的情况下，能够熟练地使用工作当然是好啊，但是如果相反就本末倒置了。*之所以说是死的是它的操作是既定的，它并不会给你带来解决方案，只是一个工具。*

也许…

所以，我扯了那么多，你清楚怎样才能胜任产品经理的工作吗？

产品经理的工作是要具备某些特质的人才能够胜任的，而检验你是否具备这些特质，且可以把这些特质运用到怎样的高度和深度，就直接投身到某一产品开发中，在系统的开发中去思考和感受，你就知道了，面试的时候也就有话可说了。

其他的特质或者其他技能不是说不要有…有总是更好的，而且某些公司也会明确规定要求你有什么技能，比如web开发？APP开发？

所以最后我想说…祝福我自己…！并不是人人都是产品经理，想当一个合格甚至是优秀的产品经理并没有理所当然的简单…🍭

那在我心中怎么算是一个合格的产品经理（平台型）呢？

首先产品经理定是要热爱产品并享受团队合作的，而平台型产品经理是产品经理的一个细分。在我看来，平台型产品经理需要从0到1打造产品，从【需求挖掘与分析到开发技术与实施性的讨论，再从产品功能路径设计到功能的实现】。这期间需要有抓住问题核心的能力、较强的思维逻辑能力，以及懂得或者能够理解开发技术的能力。同时，在产品打造过程中，需要有较强沟通能力和抗压能力，能够协调各小组的工作并能引导各小组在合作中达成共识，高效并愉快地进行产品开发。在产品打造后或者说产品上线后，注重观察产品使用情况并进行分析和判断，做进一步的产品优化和迭代。

而我在华师匣子的开发过程中所做的事情就是从0到1开发产品，并且在产品开发中协调各小组的工作，及时发现和处理开发问题，并更新产品开发进度。期间，建立了对产品开发的整体认识并提升了自己的协作沟通能力和抗压能力，并发现了自己热爱打造产品及享受其中的团队协作。
